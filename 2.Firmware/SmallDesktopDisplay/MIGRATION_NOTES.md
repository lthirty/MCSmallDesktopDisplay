## 2. 当前已固化的关键配置（避免白屏）
工程通过 `platformio.ini` 的 `build_flags` 固化了以下参数：
- 驱动：`ST7789_2_DRIVER`
- 分辨率：`240x240`
- 引脚：`TFT_CS=15`、`TFT_DC=0`、`TFT_RST=2`、`TFT_BL=5`
- 频率：`SPI_FREQUENCY=27000000`

迁移目录后，只要 `platformio.ini` 还在，这些配置就会自动生效。

## 3. 在新目录继续构建/烧写
在工程根目录执行：

```powershell
& "$env:USERPROFILE\.platformio\penv\Scripts\platformio.exe" run
& "$env:USERPROFILE\.platformio\penv\Scripts\platformio.exe" run -t upload --upload-port COM3
```

若串口不是 `COM3`，改成实际端口（如 `COM4`）。

也可直接用脚本（已自动处理长路径）：

```powershell
powershell -ExecutionPolicy Bypass -File .\tools\pio_build_upload.ps1 -Port COM3
```

## 4. 长路径处理（Windows）
若目录过长导致库安装或构建异常，使用短盘符临时映射后再编译：

```powershell
cmd /c subst X: "你的工程目录"
& "$env:USERPROFILE\.platformio\penv\Scripts\platformio.exe" run -d X:\
& "$env:USERPROFILE\.platformio\penv\Scripts\platformio.exe" run -d X:\ -t upload --upload-port COM3
cmd /c subst X: /d
```

## 5. 建议流程
1. 先 un` 编译通过。
2. 关闭串口监视器后再 un -t upload`。
3. 如遇 `COM` 端口占用，先关闭 PlatformIO/串口助手中的监视窗口再重试。
