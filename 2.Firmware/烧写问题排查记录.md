# 固件编译烧写问题记录（2026-04-15）

## 1. 烧写后黑屏

### 现象
- 固件烧写成功，但屏幕无显示（黑屏）。

### 根因
- 初次烧写使用了 `platformio.ini` 的默认环境 `esp12e`，该配置里缺少 TFT 屏幕驱动和引脚宏（如 `ST7789_2_DRIVER`、`TFT_CS/TFT_DC/TFT_RST/TFT_BL` 等）。
- 项目真正用于屏幕显示的配置在 `platformio_display.ini`（`nodemcuv2` 环境）中。

### 解决办法
1. 使用显示专用配置烧写：
   - `pio run -c platformio_display.ini -e nodemcuv2 -t upload --upload-port COM3`
2. 为避免后续再次烧错，已把 `platformio.ini` 同步为可直接用于屏幕的配置（加入显示相关 `build_flags`、`upload_port=COM3`、`extra_scripts=post:tools/export_bin.py`）。

### 结论
- 重新按显示配置烧写后，流程成功完成，黑屏风险点（配置错误）已消除。

## 3. 上传失败：COM3 被占用

### 现象
- 编译成功但上传失败：`could not open port 'COM3': PermissionError(13, '拒绝访问。')`。

### 根因
- 串口监视器或残留 `pio/python` 进程占用 COM3。

### 解决办法
1. 关闭占用进程：
   - `Get-Process pio | Stop-Process -Force`
2. 再次上传：
   - `pio run -e esp12e -t upload`

### 结果
- 释放端口后上传成功。

## 4. 编译告警（非阻断）

### 现象
- 存在多条 warning，例如：
  - `missing terminating " character`（`SmallDesktopDisplay.ino:1192` 等）
  - `SPIFFS has been deprecated`
  - `suggest parentheses around '&&' within '||'`

### 影响判断
- 当前均为告警，不阻断编译和烧写。

### 建议处理
1. 将多行 HTML 字符串改为原始字符串（`R"(...)"`）或确保反斜杠续行符后没有空白字符。
2. 逐步从 `SPIFFS` 迁移到 `LittleFS`。
3. 给复杂布尔表达式补括号，提升稳定性和可读性。
